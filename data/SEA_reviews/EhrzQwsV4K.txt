 **Summary:**
The paper introduces Code-L2MAC, a novel framework that integrates large language models (LLMs) with an external memory system to enhance code generation capabilities. This system, inspired by stored-program computers (SPCs), utilizes a control unit to manage the context window of the LLM, ensuring that it remains within the model's memory constraints. The framework is designed to handle complex code generation tasks by breaking them down into manageable sub-instructions, which are then executed using the LLM. The system also includes a mechanism for error detection and correction, which helps ensure the quality of the generated code. The paper demonstrates the effectiveness of Code-L2MAC through experiments on various system design tasks, showing its potential to generate large and complex code bases.

**Strengths:**
- The paper introduces a novel framework, L2MAC, which integrates large language models (LLMs) with external memory to generate large and complex code bases, addressing the limitations of fixed context windows in LLMs.
- The framework is designed to handle complex code generation tasks by breaking them down into manageable sub-instructions, which are then executed using the LLM.
- Code-L2MAC, an application of L2MAC for long code generation, is shown to outperform existing solutions by generating large and complex code bases.
- The paper is well-written and provides a comprehensive evaluation of the proposed method, demonstrating its effectiveness in generating large and complex code bases.
- The framework includes mechanisms for error detection and correction, which are crucial for ensuring the quality of the generated code.

**Weaknesses:**
- The paper lacks clarity in some sections, particularly in the descriptions of the memory store and the control unit, which could benefit from more detailed explanations or diagrams.
- There is a lack of discussion on the limitations of the proposed framework, such as the scalability and complexity of the code generation tasks it can handle.
- The paper does not compare its approach with other existing memory-augmented LLMs or reflecting LLMs, which could provide a more robust evaluation of the proposed method.
- The experimental setup and the details of the evaluation metrics used are not sufficiently detailed, which could limit the reproducibility and transparency of the results.
- The paper does not provide a detailed discussion on the potential negative societal impacts of the proposed framework.

**Questions:**
- Can you clarify how the memory store is implemented in practice? Is it a simple list of strings, or does it have more complex features?
- How does the control unit manage the interaction between the LLM and the memory store? Is there a specific mechanism for updating or retrieving information from the memory?
- How does the proposed framework handle the scalability and complexity of the code generation tasks it can handle? What are the limitations in terms of the size and complexity of the code bases it can generate?
- Could you provide more details on the evaluation metrics used in the experiments? How are the features in the "Features %" metric defined, and how are they evaluated?
- How does the proposed framework compare with other existing memory-augmented LLMs or reflecting LLMs in terms of performance and functionality?
- How does the proposed framework ensure the quality and correctness of the generated code? What are the specific mechanisms for error detection and correction?
